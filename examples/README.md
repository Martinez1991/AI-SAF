# Examples

Worked assessments applying AI-SAF to concrete architectures. Each shows the target
level, a subset of requirements, the verification performed, and the result — so
adopters see *how* to use the standard, not just read it.

- [`internal-prototype-l1.md`](internal-prototype-l1.md) — internal doc-Q&A prototype at L1 (partial).
- [`rag-chatbot-l2.md`](rag-chatbot-l2.md) — customer-facing RAG chatbot at L2 (partial).
- [`tool-using-assistant-l2.md`](tool-using-assistant-l2.md) — internal tool-using assistant at L2 (partial).
- [`multi-agent-workflow-l3.md`](multi-agent-workflow-l3.md) — multi-agent financial-operations workflow at L3 (partial).

These map to the four deployment archetypes in [`../threat-models/README.md`](../threat-models/README.md).
